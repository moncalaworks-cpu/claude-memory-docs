Before committing changes:

```bash
npm test
```

This runs all validation checks. If any fail, the output will show which files need fixes.

## ğŸ“‹ Common Tasks

### Adding a New Documentation File

1. Create the `.md` file with content
2. Add a reference to `MEMORY.md` under appropriate section
3. Run `npm run update-index` to update the inventory
4. Commit and push

The auto-index workflow will verify everything is correct.

### Fixing Broken Links

1. Run `npm run validate-links` to identify broken links
2. Update the file paths in the reference
3. Test: `npm run validate-links` should pass

### Checking for Orphaned Files

1. Run `npm run check-structure`
2. Either add references to `MEMORY.md` or delete unused files
3. If deleting files, commit the removal
